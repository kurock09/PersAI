# Skills Growth Playbook — RU B2B Wave 1

> Рабочий документ founder/growth-стратегии по приоритизации и наполнению Skills (наших "AI-wrapper" продуктов внутри PersAI). Это НЕ архитектурный ADR — ADR не требуется, пока мы используем уже существующие механизмы Skills / Knowledge Cards / Documents / Scenarios (ADR-079, ADR-118, ADR-119, ADR-136). Если в процессе понадобится расширять саму механику сценариев (новые типы шагов, ветвление, триггеры) — тогда стоп, отдельный аудит + мини-ADR, не раньше.

**Статус:** Wave 1 — все три skill полностью занесены через MCP и подтверждены `chat_smoke` (2026-07-05). Skill #3 заменён founder 2026-07-05 с «Коммерческие предложения и B2B-продажи» на «Финансовая аналитика и управленческий учёт для малого бизнеса» — генератор документов КП не совпадает с сильными сторонами платформы (работа с документами/таблицами/аналитикой — да, holistic sales copywriting — нет).

- **Skill #2 «Юридические черновики»** (id `1cf5ca25-0cac-411a-a4d4-18c2535023e2`) — 7 knowledge cards + PDF fundamentals + сценарий `draft_contract_or_claim`. `chat_smoke`: документ (DOCX-договор) реально сгенерирован и доставлен в чат (проверено через прямой `curl` к `/workspace-files` + `chat_fetch_attachment`, извлечён и сверен текст). Первый вызов `chat_smoke` вернул клиентский `fetch failed` при генерации документа, но серверная генерация прошла успешно — задержка на стороне SSE/long-poll, не баг сценария. **PASS.**
- **Skill #3 «Финансовая аналитика»** (id `288cd7f8-503f-4249-a853-cc10111a3f81`, репофрейминг seed `sales-coach`) — 8 knowledge cards + PDF fundamentals + сценарий `financial_health_check`. `chat_smoke` (5 ходов с `surfaceThreadKey` для непрерывности треда): корректно определил тип запроса (кассовый разрыв), построил P&L и cash-flow разбор с показанными формулами, дал точный расчёт необходимого буфера (734 000 ₽ при дефиците −674 000 ₽), сравнил маржу с отраслевым бенчмарком из PDF fundamentals (68.8% валовая / 60.1% чистая vs диапазон 60-85%/15-30% для IT-услуг), дал 3 конкретные рекомендации без нарушения guardrails (не давал точных налоговых советов, направил к skill «Бухгалтерия и налоги»), и чисто завершился (`chatSkillState.status: inactive`, все 4 todo `completed`). **PASS.**

**Важные процессные уроки:**
- `chat_smoke` без `surfaceThreadKey` создаёт новый чат на каждый вызов (нет продолжения диалога) — для многоходовых сценарных тестов обязательно передавать одинаковый `surfaceThreadKey` во всех вызовах одного теста.
- Инструмент `skill_scenario_upsert` (и `skill_card_upsert`/`skill_upsert`) в `@persai/admin-mcp` принимал `body` как непрозрачный `z.record(z.unknown())` — из-за этого во время сессии было три реальных 400-ошибки (`scenarioKey` в body вместо top-level `key`, `title`/`instructions` вместо `steps[].directive`, локале-объект для card `title` вместо простой строки). **Исправлено в этой же сессии** — типизированные Zod-схемы в `packages/persai-admin-mcp/src/server.ts`, зеркалящие точные API-парсеры; typecheck/lint/test зелёные, `dist` пересобран. Живому MCP-подключению `user-persai` в Cursor нужен reload, чтобы подхватить новую схему.

## 1. Контекст и вводные от founder (2026-07-05)

- Рынок: **RU-first**, английский — потом.
- Канал монетизации первой волны: **web-chat** (текущий продукт).
- Модель монетизации по skill: **пока не определена** — founder сознательно откладывает решение ("потом введём тарифы на Skill"). Сейчас цель — не пилотный биллинг, а контент и качество сценария. Все skill пока внутри существующих тарифов/квот.
- Объём Wave 1: **3 skill**, максимальная глубина и полировка (текст KB + PDF fundamentals + сценарий), прежде чем расширяться дальше.
- Авторство контента баз: **agent пишет всё сам** (текстовые карточки + PDF fundamentals) на основе исследования и лучших практик.
- **Ключевой стратегический сигнал от founder:**
  - Спрос от маркетплейс-продавцов (карусели, аватар-видео) есть, но это низкочековый сегмент ("они бомжи") — **сознательно не идём в эту нишу первой волной**, несмотря на то что инфраструктура (image/video generate, series/carousel) это уже отлично умеет.
  - Нужно смотреть в сторону **более денежного B2B**.
  - **РФ блокирует почти все западные AI-инструменты (ChatGPT, Claude, Gemini и т.д.)** для обычного доступа (нужен VPN + часто иностранная карта). Это огромное структурное преимущество дистрибуции для легального доступного российского AI-продукта — конкурент "просто открой ChatGPT" физически недоступен большинству целевой аудитории без костылей.
- Traction: есть небольшая тестовая/бета-база пользователей (`some_users`) — реальных данных по использованию skill пока недостаточно для приоритизации по фактам, полагаемся на рыночный анализ + здравый смысл founder.

## 2. Рыночное исследование (2026, сводка)

Источники: разбор профильных статей 2026 г. про AI SaaS/wrapper-ниши, отчёты Sensor Tower / Appfigures / Statista по доходам AI-чат-приложений, разборы монетизации AI-компаньонов (Replika/Character.AI/Chai).

Ключевые выводы:

1. **Голые "обёртки" (thin wrappers) массово проваливаются.** ~90% AI wrapper-стартапов не взлетают, маржа 25–35% против 70–85% у обычного SaaS — потому что как только базовая модель становится лучше, "продукт" обесценивается. Побеждают вертикальные продукты с собственным контентом/данными как рвом (defensibility) — это ровно то, что у нас: текстовая KB + PDF fundamentals + сценарии = proprietary content moat поверх модели.
2. **Самые прибыльные направления 2026 (по WTP и defensibility):** юридический документооборот, комплаенс/регуляторика, бухгалтерия/финансовая отчётность, недвижимость (контент/оценка), e-commerce контент, sales/proposal-документы (КП, RFP), техническая документация. Общий паттерн — **выше ставка/риск ошибки → выше готовность платить.**
3. **Самый сильный драйвер продаж — прямая связь с деньгами или риском:** "если твой софт заработает клиенту $1000, он не задумываясь заплатит $100"; и второй по силе — **страх штрафа/комплаенса** (регуляторика, отчётность).
4. **AI-компаньон/коучинг ниши (диетолог, психология, и т.п. — наш personal-блок) монетизируются очень хорошо в мире** (Replika ~$200M/год, Character.AI ~$150M/год) через подписку + token-upsell, но: (а) требуют много органического/creator-based трафика (paid-реклама в этой категории часто банится), (б) это не то, на чём фокусируется founder сейчас (B2B приоритет). **Оставляем как Wave 2+ трек**, не Wave 1.
5. **Для РФ есть уникальный дополнительный вектор**, которого нет в англоязычных источниках: блокировка западных AI = структурное преимущество дистрибуции. Это применимо и к B2C, и к B2B, но у B2B ещё добавляется **корпоративный запрет на использование зарубежных AI-инструментов с рабочими данными** (комплаенс/безопасность) — второй независимый повод выбрать российский продукт, не только "не могу зайти", но и "не имею права".

## 4. Wave 1 — три skill в глубину

Общий персонаж (ICP) для всех трёх: **владелец/директор ИП или ООО микро- и малого бизнеса в РФ**, без штатного бухгалтера/юриста/отдела продаж, который физически не может (или не хочет разбираться, как) зайти в ChatGPT. Это осознанно один и тот же ICP для всех трёх skill — так легче собирать общую воронку/апселл в будущем (когда появятся тарифы по skill).

Каждый skill ниже структурирован по требованию founder:
- **Текстовая база (Knowledge Cards)** — короткие, быстрые факты для "wow"-эффекта с первого вопроса.
- **PDF fundamentals** — фундаментальные знания, которые агент читает ПЕРВЫМ через `knowledge_fetch`/skill retrieval, чтобы не "гулял по интернету", а видел специфику своего skill.
- **Сценарий** — короткий, чёткий поток, ведущий user к идеальному результату за минимум шагов.

### 4.1 Skill: «Бухгалтерия и налоги для ИП/ООО»

**Существующий seed:** `accounting-tax` (work). Переиспользуем key, полностью пересобираем наполнение.

**Обещание (wow с первого сообщения):** "Скажи, кто ты (ИП/ООО, доход, сотрудники) — за один ответ получишь: правильный налоговый режим + персональный календарь дедлайнов."

**Text Knowledge Cards (короткие, атомарные):**
1. УСН «Доходы» vs «Доходы минус расходы» — как выбрать (сравнительный чек-лист, точка безубыточности по доле расходов).
2. Календарь отчётности и платежей ИП/ООО на текущий год (кварталы, даты).
3. Фиксированные страховые взносы ИП + как уменьшить налог УСН на взносы (для "Доходы").
4. Топ-10 ошибок начинающих ИП с налогами (кассовый разрыв, забытые авансовые платежи, и т.д.).
5. Когда пора менять режим — пороги дохода/сотрудников для УСН/патента/НПД.
6. ИП vs ООО — разница в налогах и ответственности учредителя.
7. НДС для малого бизнеса — когда возникает обязанность, лимиты освобождения.
8. Самозанятость (НПД) — когда выгодна, ограничения по доходу и видам деятельности.

**PDF Fundamentals (глубокий справочник, читается агентом первым):**
- Полное руководство по налоговым режимам РФ для малого бизнеса (УСН, ОСН, ПСН, НПД) — ставки, лимиты применения, плюсы/минусы, кто может применять.
- Календарь отчётности с формами (декларация УСН, РСВ, 6-НДФЛ, статистика) — по кварталам, с датами и штрафами за просрочку.
- Справочник по страховым взносам (фиксированные + 1% с превышения 300 000 ₽) с формулами расчёта.
- 5–8 разобранных кейсов расчёта налога для типовых сценариев дохода/расходов (услуги, розница, с сотрудниками/без).
- Ссылки/пометки, где данные могут устареть (ставки/пороги меняются ежегодно) — с явным warning, что перед платежом agent рекомендует свериться с ФНС/1С, а не выдаёт себя за 100% актуальный источник.

**Сценарий (steps):**
1. Уточнить форму (ИП/ООО), текущий/планируемый налоговый режим, регион, есть ли сотрудники, примерный годовой доход/расход.
2. Определить/подтвердить оптимальный налоговый режим (сверка с fundamentals PDF, таблица порогов).
3. Выдать персональный календарь дедлайнов (налоги, взносы, отчёты) на ближайший год.
4. Предложить конкретное следующее действие (например: зафиксировать дедлайны как reminders/задачи, подготовить пояснение к декларации, посчитать текущий налог).
5. Exit condition: пользователь получил налоговый режим + календарь + понятный следующий шаг.

### 4.2 Skill: «Юридические черновики для малого бизнеса»

**Существующий seed:** `legal-drafting` (work). Пересобираем.

**Обещание:** "Опиши сделку/ситуацию — получи черновик договора/претензии за один сценарий, с пометкой рискованных мест."

**Text Knowledge Cards:**
1. Чек-лист обязательных пунктов в договоре оказания услуг.
2. Оферта vs договор — когда что использовать.
3. NDA — минимальный набор условий, которые реально защищают.
4. Как оформить претензию контрагенту по просрочке оплаты (структура, сроки ответа).
5. **Договор с самозанятым — как не получить переквалификацию в трудовые отношения** (частая и дорогая ошибка малого бизнеса — штрафы + доначисление взносов).
6. Топ-5 юридических рисков ИП при работе с подрядчиками/фрилансерами.
7. Что проверить перед подписанием любого договора (чек-лист red flags).

**PDF Fundamentals:**
- Библиотека шаблонов договоров (оказание услуг, поставка, агентский, NDA, с самозанятым) — с построчными пояснениями по каждому существенному условию.
- Гид по договорной работе малого бизнеса: от согласования условий до подписания и хранения.
- Претензионно-исковая работа: шаблоны претензий, обязательные сроки досудебного порядка, что делать при отказе.
- Явный disclaimer (как в seed guardrails): черновики — не замена юриста для сложных/высокорисковых сделок, рекомендация проверки у юриста при крупных суммах.

**Сценарий:**
1. Определить тип документа (договор/претензия/NDA/др.) и стороны сделки.
2. Собрать ключевые условия через уточняющие вопросы (предмет, сумма, сроки, ответственность, особые условия).
3. Сгенерировать черновик документа (через `document` tool → DOCX) с инлайн-пометками рискованных мест.
4. Выдать чек-лист "что проверить перед подписанием" + предложить сохранить/отправить (`files.attach`).
5. Exit condition: пользователь получил готовый черновик + понимание рисков этого конкретного документа.

### 4.3 Skill: «Финансовая аналитика и управленческий учёт для малого бизнеса»

**Существующий seed:** `sales-coach` (work), слот заменён founder 2026-07-05. Полностью пересобираем под финансовую аналитику — это прямое попадание в сильные стороны платформы (документы, таблицы, аналитика), в отличие от генерации КП/копирайтинга.

**Обещание (wow с первого сообщения):** "Дай мне цифры за месяц (доходы/расходы/остатки) — получи управленческий отчёт (P&L, cash flow, unit-экономика) + 2-3 конкретных вывода, что делать дальше."

**Text Knowledge Cards:**
1. P&L (отчёт о прибылях и убытках) для малого бизнеса — что входит, простая структура на 10 строк.
2. Cash flow vs прибыль — почему бизнес может быть "прибыльным на бумаге" и без денег в кассе (кассовый разрыв).
3. Unit-экономика — как посчитать себестоимость и маржу на единицу товара/услуги.
4. Точка безубыточности — простая формула + как её использовать для решений о цене/объёме.
5. Топ-7 финансовых метрик, которые должен знать владелец малого бизнеса (маржинальность, ДДС, оборачиваемость, ROI и т.д.).
6. Как построить платёжный календарь и избежать кассового разрыва.
7. Управленческий учёт vs бухгалтерский — зачем вести отдельно от налоговой отчётности.
8. Топ-10 ошибок малого бизнеса в управлении финансами (смешение личных и бизнес-денег, отсутствие резерва, и т.д.).

**PDF Fundamentals:**
- Полное руководство по управленческому учёту для малого бизнеса: P&L, ДДС (cash flow statement), баланс в упрощённом виде — с шаблонами таблиц и построчными пояснениями.
- Методология расчёта unit-экономики и точки безубыточности — с формулами и 5–8 разобранными кейсами (услуги, розница, производство).
- Справочник финансовых метрик/коэффициентов с формулами, нормальными диапазонами по отраслям и интерпретацией "что делать, если метрика плохая".
- Гид по построению платёжного календаря и управлению кассовыми разрывами (сценарии: сезонность, отсрочки платежей клиентов, кредиторка).
- Explicit disclaimer: управленческая аналитика — не замена бухгалтерского/налогового учёта и не инвестиционная консультация; для юридически значимой отчётности — бухгалтер/аудитор.

**Сценарий:**
1. Уточнить тип бизнеса, период (месяц/квартал), доступные данные (доходы, расходы по категориям, остатки на счетах, дебиторка/кредиторка) — попросить прислать цифры текстом или файлом (Excel/CSV/фото таблицы).
2. Построить P&L за период (документ/таблица) — выручка, себестоimость, операционные расходы, прибыль, маржинальность.
3. Оценить cash flow / риск кассового разрыва на ближайший период (платёжный календарь при наличии данных о дебиторке/кредиторке).
4. Дать 2–3 конкретных, специфичных для этих цифр вывода/рекомендации (например: "маржа по товару X ниже точки безубыточности", "через 3 недели кассовый разрыв, если клиент Y не заплатит").
5. Exit condition: пользователь получил структурированный финансовый отчёт (документ/таблица) + конкретные, персонализированные выводы, готовые к действию.

## 5. Осознанно исключено / отложено (и почему)

- **Маркетплейс-продавцы (карусели/аватар-видео для WB/Ozon)** — низкий чек, генерика, конкуренты быстро копируют. Инфраструктура (image/video series) готова, но экономика невыгодна для Wave 1. Можно вернуться как Wave 3+ дешёвый top-of-funnel продукт, не как флагман.
- **Engineering-блок** (`software-engineer`, `devops-sre`, `data-analyst`, `qa-engineer`, `security-reviewer`, `ux-ui-designer`, `electronics-engineer`) — целевая аудитория (разработчики) чаще всего уже имеет обходы блокировок (VPN, self-host, корпоративный доступ) и уже пользуется Cursor/Copilot/etc. Слабый РФ-барьер-ров, сильная глобальная конкуренция. Не первая волна.
- **Personal/companion блок** (`dietitian`, `fitness-coach`, `sleep-coach`, `mental-wellbeing`, `personal-finance`, `travel-planner`, `style-assistant`, `home-household`, `parenting-assistant`, `pet-care`) — монетизируется очень хорошо в мире (подписка+token), и РФ-барьер тоже применим (обычный потребитель ещё меньше, чем бизнес, будет настраивать VPN+иностранную карту ради ChatGPT). **Кандидат на Wave 2**, но founder сейчас фокусируется на B2B — не трогаем в Wave 1.
- **Тендеры/госзакупки (44-ФЗ/223-ФЗ)** — высокая ставка и высокий спрос, но контент дорогой и рискованный (ошибка в заявке стоит контракта) — кандидат №1 для Wave 2 после обкатки процесса на Wave 1.

## 6. Процесс занесения в систему через MCP (`persai` admin MCP, ADR-136)

Для каждого skill из Wave 1, по порядку:

1. `skill_upsert` — создать/обновить core-поля + `instructionCard` (используем существующий `key`, category `work`, обновляем `body`/`guardrails`/`examples` под новый фокус).
2. `skill_card_upsert` — по одной карточке на каждый пункт из списка Text Knowledge Cards выше (`lifecycleStatus: active`, `provenanceKind: manual`).
3. `skill_document_upload` — загрузить PDF fundamentals (сначала пишем PDF локально, затем грузим через `filePath`).
4. `indexing_wait` — дождаться `ready` по всем документам/карточкам перед тестом.
5. `skill_scenario_upsert` — завести сценарий (`status: active`) с шагами из раздела 4.
6. `assistant_skills_assign` — назначить skill активному ассистенту founder (merge, не перезаписывая остальные).
7. `assistant_publish` — опубликовать (Anthropic workflow: publish перед каждым новым тестом сценария).
8. `chat_smoke` с конкретным `goal` (например: *"пользователь ИП на УСН 6% без сотрудников, доход 1.2 млн/год — должен получить рекомендацию режима + календарь дедлайнов за один сценарий"*) — итеративно улучшать карточки/PDF/сценарий, пока результат не идеальный.
9. Полировка: смотреть `skillActivation`, `toolSignals`, `plan.todos` (см. `packages/persai-admin-mcp/SMOKE-AGENT.md`), при необходимости добавлять недостающие карточки/шаги.

## 7. QA чек-лист на каждый skill перед "готово"

- [ ] Skill активируется по естественному запросу без явного упоминания названия skill.
- [ ] Первый ответ агента даёт "wow" — конкретный, специфичный факт/цифру/дедлайн, а не общие слова.
- [ ] Агент явно опирается на PDF fundamentals (не выдаёт устаревшие/неточные данные "из головы" общей модели).
- [ ] Сценарий укладывается в 3–5 шагов до результата, без лишних уточняющих вопросов.
- [ ] Результат — конкретный артефакт (документ/календарь/КП), который можно скачать/использовать сразу.
- [ ] Guardrails на месте (disclaimer "не замена бухгалтера/юриста для сложных случаев" — юридическая безопасность).
- [ ] `chat_smoke` с 2–3 разными вводными (разный доход/тип бизнеса) даёт стабильно хороший результат, не только на "счастливом пути".
